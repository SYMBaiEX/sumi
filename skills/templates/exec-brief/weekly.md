# Template: Exec Brief — Weekly

Terse C-suite/exec weekly brief. Numbers first, decisions second, risks flagged. Every section is optional except TL;DR and Metrics. Assume the reader has 90 seconds.

## Structure

```
*{{display_name}} — {{start_date}} → {{end_date}}*

*TL;DR*
{{One sentence. The single thing the exec must know this week.}}

*📈 Metrics (deltas only)*
• {{Metric — Δ vs. last period, absolute, context one line.}}

*✅ Decisions made*
• {{Decision. Owner. One-line rationale.}}

*🚩 Risks*
• {{Risk. Severity (P0/P1/P2). Mitigation or "needs decision".}}

*🔄 Shipped*
• {{What shipped, to whom, impact.}}

*🙋 Asks of the exec*
• {{Thing the exec needs to do or decide — ideally one.}}

*🔜 Next week watchlist*
• {{One or two things likely to need exec attention.}}
```

## Rules

- **Numbers in the first line of every bullet** where a metric exists.
- **No bullet longer than 15 words.**
- **No narrative.** Ever. If you need narrative, put it in a linked doc and reference.
- **No process language.** "Sprint planning," "standup," "OKR review" are noise to execs.
- **Asks are one ideally, three max.** Execs don't read lists of asks.
- **Trim ruthlessly.** If a section has nothing newsworthy, omit it.
