# Template: Team Update — Cross-Functional

An update for readers spanning multiple teams/functions (e.g. a weekly note to eng + product + design + support). Emphasizes dependencies, decisions that crossed team lines, and cross-team asks.

## Structure

```
*{{display_name}} — {{start_date}} → {{end_date}}*

*TL;DR*
{{One sentence readable by any function.}}

*🏆 Wins (with the team that owned each)*
• {{Outcome — *Owner: team name*}}

*🔗 Cross-team decisions*
• {{Decision that crossed team lines this period. Who agreed, what was decided, link.}}

*🛠️ In progress — dependencies flagged*
• {{Work in flight — *Depends on: team/person*}}

*🚧 Cross-team blockers*
• {{Blocker spanning teams, who owns unblocking. If none, "_Nothing this period_".}}

*📅 Meetings & decisions*  ← only if Calendar is a source
• {{Cross-team meeting with outcome.}}

*📣 External signal*  ← only if Gmail is a source
• {{Customer or external signal that affects more than one team.}}

*📈 Metrics that moved*
• {{Shared metrics — name the team that drives each.}}

*🙋 Asks (by target team)*
• {{Ask — *For: team name*}}

*🔜 Next period commitments (by team)*
• {{Team A: commitment}}
• {{Team B: commitment}}
```

## Rules

- **Every bullet names a team.** Readers scan for their team's name first.
- **Surface dependencies explicitly.** Cross-functional updates exist to make dependencies visible.
- **Neutral vocabulary.** Don't use jargon from one team (e.g. engineering sprint terms) without translation.
- **Respect time.** Cross-functional audiences are the most time-constrained — skimmable beats thorough.
